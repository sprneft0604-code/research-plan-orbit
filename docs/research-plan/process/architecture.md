# Architecture Process

Use this process when design choices matter.

## Current State

- Review existing architecture.
- Identify conventions and reusable patterns.
- Note technical debt only when it affects the request.

## Requirements

Capture:

- functional requirements
- non-functional requirements
- integration points
- data flow
- constraints

## Options

For each meaningful option, record:

- benefits
- drawbacks
- risks
- alternatives
- recommendation

## Decision

State the recommended option and why it best fits the request.

Prefer simple, maintainable designs that follow existing project patterns.
Avoid speculative complexity.
