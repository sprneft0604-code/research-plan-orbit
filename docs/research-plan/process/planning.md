# Planning Process

A valid implementation plan includes:

- overview
- requirements
- affected files or components
- phased implementation steps
- dependencies
- risk level
- testing strategy
- success criteria

Each implementation step should include:

- action
- target files or components
- reason
- dependencies
- risk

Large work should be split into independently deliverable phases.

Prefer plans that enable incremental verification.
Avoid plans where all phases must be completed before anything works.
